Key Features

Real-time Face Mood

Client-side landmark heuristics (MediaPipe) analyze expression intensity to determine emotion quickly and privately without sending images to the server.

Voice Emotion

WebAudio RMS, pitch and spectral features — heuristics plus optional TF models map audio to emotion classes for low-latency detection.

Smart Recommendations

Server adapters query YouTube & Spotify when configured. Otherwise, intelligent local fallbacks and mood-to-query mapping keep suggestions relevant.

Seamless Playback

YouTube/Spotify embeds are used where possible. When embeds are blocked, graceful fallbacks open content without breaking the user experience.

How it works

  1. 1
    Capture

    Webcam and microphone capture streams after user permission. All local by default.

  2. 2
    Analyze

    Face landmarks + audio features are computed client-side. Heuristics estimate mood.

  3. 3
    Recommend

    Frontend sends mood to server; server maps mood to queries and fetches tracks when configured.

  4. 4
    Play

    Playable embeds are inserted; if embeddable checks fail, we open watch/open pages or curated fallbacks.

Deep Dive

Face heuristics

The face module uses MediaPipe FaceMesh landmarks and geometric heuristics to compute smile, mouth openness and eye openness. This lightweight approach is fast on mobiles and avoids sending facial images to servers.

Audio features

voice.js leverages WebAudio API: RMS for energy, autocorrelation for pitch estimate, and frequency bins for spectral centroid — used in mapping to emotional categories.

Server & APIs

Flask endpoints provide search and recommendation logic. YouTube & Spotify keys (if set) enable richer server-side recommendations and embeddable playback checks.

Privacy

By default, only mood labels are sent to the server — not raw audio or video. This design reduces data leakage and makes it suitable for demos and internal prototypes.
